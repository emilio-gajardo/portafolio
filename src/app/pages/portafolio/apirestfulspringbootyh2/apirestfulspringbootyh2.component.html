Características del sistema

Fecha de desarrollo Abril 2022

Framework Frontend N.A.
Framework Backend Spring Boot 2.5.6

Contexto
  • Desarrollo de API Restful que realiza CRUD sobre BD H2.
  • Uso de Postman para las pruebas de endpoints.
  • Integración de:
    • API Java Faker como cliente para cargar BD H2.
    • AOP (Programación orientada a objetos).
    • JPA (Java Persistence Api).
    • Spring Data para uso de JPA y JDBC para H2.
    • Custom Querys.
    • Actuator para monitorización de métricas del sistema.
    • Prometheus para uso de métricas.
    • Grafana usando el datasource de Prometheus.
    • Dashboard en Grafana.
    • Swagger y Swagger Ui 2.9.2 para documentación de la Api.
    • Spring Caching para almacenar datos en el cache.
    • Redis, para almacenar el cache en un sistema externo.
    • Redisson, para conexión con Redis y manipulación de valores.
    • Spring Security, para validar a los usuarios que se conecten a la Api.
    • meta anotación sobre Pre y PostAuthorize.
    • Apache Kafka para análisis de transmisión de datos y balanceo de carga.
    • Kafka produce y Kafka consumer.
    • Lombok.

Propuesta de Solución

Se presenta esta propuesta que permite a los usuarios visualizar los registros desde la BD.
El sistema restringe el acceso solo a usuarios registrados.


Objetivos del sistema
  1. Facilitar el acceso a la información.
  2. Restringir el acceso de los usuarios.
  3. Maximizar el rendimiento del sistema.
  4. Manejo de errores y excepciones.
  5. Paginación de resultados.
  6. Diseño escalable.
  7. Alta seguridad.
  8. Facilitar la documentación de la Api.
