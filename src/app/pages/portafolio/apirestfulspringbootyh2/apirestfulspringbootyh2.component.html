Características del sistema

Fecha de desarrollo Abril 2022
Plataforma Web
Tecnologías Java 11, Spring Data, Spring Security, Spring Cache, Spring REST, Swagger, Prometheus, Grafana, Apache Kafka, Maven, Redis, Redisson
Base de datos H2
Patrón de arquitectura MVC
Patrón de diseño DAO (Data Access Object)
Metodología de desarrollo Ágil
Modelo de desarrollo Scrum
Framework Frontend N.A.
Framework Backend Spring Boot 2.5.6

Contexto
  • Desarrolla de API Restful que realizar CRUD sobre BD H2.
  • Integración de API Java Faker como cliente para cargar BD H2.
  • Integración de AOP (Programación orientada a objetos).
  • Integración de JPA (Java Persistence Api).
  • Integración de Spring Data para uso de JPA y JDBC para H2.
  • Integración de Custom Querys.
  • Integración de Actuator para monitorización de métricas del sistema.
  • Integración de Prometheus para uso de métricas.
  • Integración de Grafana usando el datasource de Prometheus.
  • Creación de Dashboard en Grafana.
  • Integración de Swagger y Swagger Ui 2.9.2., para la documentación de la Api.
  • Integración de Spring Caching para almacenar datos en el cache.
  • Integración de Redis, para almacenar el cache en un sistema externo.
  • Integración de Redisson, para conexión con Redis y manipulación de valores.
  • Integración de Spring Security, para validar a los usuarios que se conecten a la Api.
  • Integración de meta anotación sobre Pre y PostAuthorize.
  • Integración de Apache Kafka con Spring, para análisis de transmisión de datos y balanceo de carga.
  • Integración de Kafka produce y Kafka consumer.
  • Integración de Lombok.
  • Uso de Postman para las pruebas de endpoints.


Propuesta de Solución

Se presenta esta propuesta que permite a los usuarios visualizar los registros desde la BD.
El sistema restringe el acceso solo a usuarios registrados.


Objetivos del sistema
  1. Facilitar el acceso a la información.
  2. Restringir el acceso de los usuarios.
  3. Maximizar el rendimiento del sistema.
  4. Manejo de errores y excepciones.
  5. Paginación de resultados.
  6. Diseño escalable.
  7. Alta seguridad.
  8. Facilitar la documentación de la Api.
